# 阶段二：深度加工详细指令

阶段二是整个恢复流程最核心的阶段。不是简单地复制粘贴和格式化，而是像人类回忆一样对原始素材进行深度理解和重组。

## 加工策略

### 因果关系提取

不只记录"发生了什么"，更要记录"为什么"：

- ❌ 浅层：`修改了 nginx 配置`
- ✅ 深层：`因为 WebSocket 连接频繁断开，排查发现 nginx 默认超时 60s 太短，改为 300s 后解决`

### 闪光灯记忆识别（⚡ 标记）

通过以下**信号词**识别高价值记忆，标记为 `⚡`：

| 类别 | 信号词 |
|------|--------|
| 决策 | "决定"、"改为"、"选择"、"放弃"、"确认" |
| 发现 | "发现"、"原来"、"终于"、"找到了"、"根本原因" |
| 教训 | "踩坑"、"注意"、"以后"、"记住"、"千万别"、"又"、"坑" |
| 重要 | "关键"、"必须"、"重要"、"核心"、"红线" |
| 情绪 | "折腾"、"终于好了"、"搞定"、"头疼"、"太坑了" |

被标记为 `⚡` 的条目在后续所有阶段中**永不省略、永不折叠**。

### 主题聚合（替代纯时间流）

同一件事跨多轮对话的内容**归到同一个主题下**，而非散落在时间线各处：

- ❌ 按时间：`10:00 - 讨论bug` → `10:30 - 写代码` → `11:00 - 继续修bug`
- ✅ 按主题：`### 修复 WebSocket 断连问题` 下包含完整的排查→定位→修复→验证过程

### 标签和交叉引用

- 为每个主题添加标签 `#标签`，便于跨日期检索
- 如果某主题与其他日期的事件相关，添加交叉引用 `→ 参见 YYYY-MM-DD#主题`

## 信息分类

深度加工时同时识别两类信息：

- **记忆类**：写入 daily notes 和 MEMORY.md（事件、决策、发现、教训）
- **配置类**：写入 workspace 文件（用户画像、Agent 身份/人格/规则的变更）
- 对话中出现 "叫我 xxx"、"你以后 xxx"、"记住 xxx" 等指令时，尤其注意分类

## 正式 daily note 格式

```markdown
# YYYY-MM-DD Daily Notes

## 事件记录

### HH:MM - 主题摘要 #标签1 #标签2
- 背景/起因
- 关键过程
- ⚡ 重要发现或决策（附原因）
- 结果/结论
- → 参见 YYYY-MM-DD#相关主题

### HH:MM - 另一个主题 #标签
- ...

## 关键收获
- ⚡ 经验教训（附具体场景）
- 技术发现（附验证方式）

## 待办
- [ ] 未完成的事项
```

## 深度加工原则

- **追问"为什么"**：每个决策和操作都尝试还原动机
- **保留因果链**：问题→排查→原因→方案→结果，完整记录
- **标记闪光灯**：用信号词识别高价值记忆，标 `⚡`
- **按主题聚合**：同一事件的多轮对话合并到一个主题下
- **去除噪音**：寒暄、重复确认、System 元信息、纯工具输出
- **时区转换**：JSONL timestamp 为 UTC，转换为用户时区（从 USER.md 读取）
- **添加标签**：每个主题至少一个 `#标签`，便于日后检索
- **交叉引用**：发现跨日期的关联时，双向添加 `→ 参见` 链接
