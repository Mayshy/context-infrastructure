# 阶段三步骤 A：Workspace 配置文件更新规则

从 daily notes 中收集在阶段二标记的配置类信息，按路由规则更新对应 workspace 文件。

## 更新流程

1. **读取所有正式的** `memory/YYYY-MM-DD.md` 文件，收集配置类信息
2. **读取当前的 workspace 文件**（如果存在）：`USER.md`、`SOUL.md`、`IDENTITY.md`、`AGENTS.md`、`TOOLS.md`
3. **按以下规则更新对应文件**

## 各文件更新规则

### `USER.md` — 用户画像

- 提取用户自我介绍、偏好声明、习惯描述
- 信号词：`"叫我"`、`"我是"`、`"我在"`、`"我喜欢"`、`"我习惯"`、`"我的时区"`
- 更新策略：合并到已有内容，新信息覆盖旧的同类信息（如新时区替换旧时区）

### `SOUL.md` — Agent 人格/语气

- 提取用户对 Agent 说话方式的调整指令
- 信号词：`"用中文"`、`"语气"`、`"风格"`、`"像...一样"`、`"不要太"`、`"简洁"`、`"详细"`
- 更新策略：追加新指令，如果与已有指令矛盾则以更新的为准

### `IDENTITY.md` — Agent 身份

- 提取用户为 Agent 设定的身份属性
- 信号词：`"你叫"`、`"你的名字"`、`"你是"`、`"用...做你的 emoji"`、`"头像"`
- 更新策略：直接更新对应字段（Name / Creature / Vibe / Emoji / Avatar）

### `AGENTS.md` — Agent 行为规则

- 提取用户设定的行为约束和工作规范
- 信号词：`"以后"`、`"记住"`、`"不要"`、`"千万别"`、`"每次...都要"`、`"规则"`
- 更新策略：追加到规则列表，去重

### `TOOLS.md` — 工具使用指导

- 提取用户对工具使用方式的自定义要求
- 信号词：`"用 xxx 而不是"`、`"搜索时"`、`"运行时加上"`
- 更新策略：追加到对应工具章节

## 更新原则

- **文件不存在时不创建**：只更新已存在的 workspace 文件，不主动创建新文件
- **保留已有内容**：追加/合并，不清空重写
- **冲突时新优先**：同一字段有新旧值时，以 session 中更晚出现的为准
- **标注来源**：在关键更新旁标注 `(→ YYYY-MM-DD)` 便于溯源
- **不确定时跳过**：只写入用户明确表达的指令，不推测用户意图

## 路由原则速查

| 信息类型 | 目标文件 | 典型示例 |
|----------|----------|----------|
| 关于用户自己 | `USER.md` | "叫我小李"、"我在东八区" |
| Agent 说话方式 | `SOUL.md` | "回复用中文"、"语气温柔一点" |
| Agent 身份设定 | `IDENTITY.md` | "你叫 xxx"、"用 🐱 做 emoji" |
| Agent 行为规则 | `AGENTS.md` | "部署前先跑测试"、"不要动 xxx" |
| 工具使用方式 | `TOOLS.md` | "用 rg 不用 grep" |
| 具体事件和经验 | `MEMORY.md` | 经验教训、技术发现、决策记录 |
